## 작업 목표

사용자 입력 데이터의 유효성을 검증하는 모듈을 구현합니다.
음수 입력, 범위 초과, 논리적 오류 등을 검증하고 사용자에게 명확한 오류 메시지를 제공합니다.

---

## 작업 내용

### 1. 입력 데이터 검증 함수

**함수**: `validate_inputs(inputs: Dict[str, Any]) -> Tuple[bool, List[str]]`

**기능**:
- 입력 데이터 딕셔너리를 받아 모든 필드를 검증
- 검증 실패 시 오류 메시지 리스트 반환
- 검증 성공 시 True와 빈 리스트 반환

**검증 항목**:
1. **기본 정보 검증**
   - 현재 나이: 0 이상 150 이하
   - 은퇴 나이: 현재 나이보다 크고 100 이하

2. **소득 정보 검증**
   - 연봉: 0 이상
   - 연봉 증가율: 0 이상 20 이하
   - 보너스: 0 이상

3. **소비 정보 검증**
   - 월 지출: 0 이상
   - 연간 고정 지출: 0 이상

4. **자산 및 부채 검증**
   - 총 자산: 0 이상
   - 총 부채: 0 이상

5. **논리적 검증**
   - 은퇴 나이 > 현재 나이
   - 월 지출이 연봉의 합리적 범위 내인지 (선택적)

### 2. 개별 필드 검증 함수

**함수들**:
- `validate_age(age: int) -> Tuple[bool, str]`
- `validate_retirement_age(current_age: int, retirement_age: int) -> Tuple[bool, str]`
- `validate_salary(salary: float) -> Tuple[bool, str]`
- `validate_salary_growth_rate(rate: float) -> Tuple[bool, str]`
- `validate_expense(expense: float) -> Tuple[bool, str]`
- `validate_assets(assets: float) -> Tuple[bool, str]`
- `validate_debt(debt: float) -> Tuple[bool, str]`

**기능**:
- 각 필드의 유효성을 개별적으로 검증
- 검증 실패 시 오류 메시지 반환

### 3. 논리적 일관성 검증 함수

**함수**: `validate_logical_consistency(inputs: Dict[str, Any]) -> List[str]`

**기능**:
- 입력 데이터 간의 논리적 일관성 검증
- 예: 은퇴 나이가 현재 나이보다 작으면 오류
- 예: 월 지출이 연봉을 초과하면 경고 (선택적)
