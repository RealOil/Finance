# 경제적 자유 시뮬레이션 툴 - 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 프로젝트 목적

'경제적 자유'의 개념을 다양한 경제 상황(환율, 인플레이션, 연봉 인상률, 투자 시뮬레이션 등) 속에서 체험하고 계산해볼 수 있는 툴을 제공하여, 사용자가 자신의 재정 상황을 이해하고 미래를 계획할 수 있도록 돕는다.

### 1.2 프로젝트 범위

- **타겟 사용자**: 대중 (최대한 많은 사용자 대상)
- **플랫폼**: Streamlit 웹 애플리케이션 + Notion 템플릿
- **배포 단계**:
  - Phase 1: 접근 가능한 프로토타입 (웹 배포 또는 모바일 접근)
  - Phase 2: 완전한 웹 배포 (Streamlit Cloud) + Notion 템플릿 배포

### 1.3 핵심 가치

- **쉬운 접근성**: 복잡한 금융 지식 없이도 사용 가능
- **명확한 인사이트**: 전문 용어 대신 이해하기 쉬운 문장으로 결과 제공
- **시각화 중심**: 그래프와 차트를 통한 직관적 정보 전달

- **간편 입력**:
  - CSV 파일 업로드 지원 (선택)
  - 단계별 가이드 제공
  - 기본값/예시 값 제공

### 2.3 핵심 기능 (우선순위별)

#### 2.3.1 MVP 기능 (Phase 1 - 상위 2개 카테고리)

**우선순위 1: 소득 및 지출 기반 인사이트**

- 현재 소비 습관 유지 시 N년 뒤 남는 자산 추정치 계산
- 소득 대비 지출 비율 계산 및 재정 건전성 등급 평가 (A~F 등급)
- 소비 패턴 변경 시나리오별 결과 비교
  - 예: 지출 10% 감소 시 5년/10년 뒤 자산 변화
  - 예: 지출 20% 증가 시 자산 소진 시점
- 소득 증가율 vs 물가상승률 비교 (실질 소득 증감 여부)
- 연봉 인상률별 미래 자산 비교 시뮬레이션 (0%, 3%, 5%, 10%)
- **출력 형식**:
  - 그래프: 연도별 자산 변화 추이
  - 텍스트: "현재 소비 패턴을 유지하면 10년 후 약 XXX만원이 남습니다. 하지만 지출을 10% 줄이면 XXX만원 더 모을 수 있어요."

**우선순위 2: 미래 리스크 및 시나리오 경고**

- 은퇴 시점 월 지출 유지 가능 여부 계산
- 경제 위기 시나리오 적용 (자산 -30% 하락 시 생존 가능 기간)
- 소득 중단 상황 버티기 계산 (6개월, 1년, 2년)
- 주택담보대출 금리 상승 시나리오 (2%p 상승 시 월 상환액 변화)
- 장기 의료비 증가 추정 대비 자산 위험도 평가
- **출력 형식**:
  - 경고 메시지: "현재 자산으로는 소득 중단 시 약 8개월만 버틸 수 있습니다."
  - 시나리오별 대시보드
  - 위험도 점수 (0~100)

#### 2.3.2 Phase 2 기능 (추가 카테고리 - 인사이트 제공 능력 가중치 반영)

**우선순위 3: 기회비용 기반 분석** (종합 점수: 15점, 인사이트: 높음)

- 소비 → 투자 전환 시나리오 ("만약 그때 안 썼다면?")
- 특정 소비 습관 비용의 10년 누적 영향 계산
- 월 정기 투자 비교 시뮬레이션 (ETF vs 예금)
- 소비 vs 투자 미래 격차 그래프
- **인사이트**: 소비 습관 개선 동기 부여, 투자 시작 결정에 직접 활용

**우선순위 4: 부동산 기반 분석** (종합 점수: 11.5점, 인사이트: 높음)

- 과거 대비 현재 부동산 실질가치 변화
- 대출금리 변동에 따른 손익 비교
- 월세 전환 시 break-even point 계산
- 매도/보유 전략 비교 시나리오
- **인사이트**: 매도/보유 결정에 직접 활용, 대출 상환 전략 수립

**우선순위 5: 보험 및 장기 금융상품 인사이트** (종합 점수: 11.5점, 인사이트: 높음)

- 보험료 총액 vs 보장 가치 비교
- 동일 금액 투자 대비 보험 효율 평가
- 중도 해지 vs 유지 시 재정 손익
- **인사이트**: 보험 해지/유지 결정에 직접 활용, 즉시 실행 가능한 재정 결정

**우선순위 6: 경제적 체력 지표** (종합 점수: 12점, 인사이트: 중간)

- 자산 대비 부채 비율
- 순자산 성장 속도 → "성장 체력 점수"
- 목표 자산 규모까지 남은 시간 (페이스 분석)
- 지출-투자 균형 비율 최적 모델 제안
- 금융 스트레스 점수 (부채/고정비 기반)
- **인사이트**: 현재 상태 파악, 목표 달성을 위한 페이스 조정 필요성 인식

**우선순위 7: 인플레이션 기반 가치 비교 인사이트** (종합 점수: 9.5점, 인사이트: 중간)

- 현재 자산 총액의 실질가치 계산 (물가 반영)
- 최근 10년 평균 인플레이션 대비 자산 성장률
- 인플레이션만 반영 vs 실제 성장률 비교 그래프
- 미래 10년 뒤 현재 자산의 예상 구매력
- 과거 5년 전 기준 구매력 차이 계산
- 월급 인상률 vs 물가상승률 비교
- **인사이트**: 장기 계획 수립에 유용, 투자 의사결정에 도움

**우선순위 8: 목표 설계 및 전략 추천** (종합 점수: 9.5점, 인사이트: 높음)

- 현재 데이터 기반 1년/3년/5년/10년 목표 시뮬레이션
- 은퇴 시 월 생활비 유지 목표 자산 계산
- 자산 포트폴리오 예상 CAGR 계산
- 위험 성향 기반 맞춤 자산 분배 추천
- "지금 바로 해야 할 5가지 금융 행동 리스트"
- **인사이트**: 즉시 실행 가능한 액션 아이템, 장기 재정 계획 수립 필수
- **참고**: 구현 난이도 높음 (4~5주 소요)

**우선순위 9: 환율·글로벌 변수 인사이트** (종합 점수: 10점, 인사이트: 중간)

- 원화가치 하락 대비 자산 잠식률
- 해외 ETF 투자 대비 환차손/환차익 자동 계산
- 특정 시점 환율로 환전 시나리오
- **인사이트**: 해외 투자 시 참고 정보 제공

**우선순위 10: 투자 시나리오 기반 인사이트** (종합 점수: 6.5점, 인사이트: 중간)

- 과거 특정 시점 투자 시 현재 가치 계산
- 다양한 자산군 비교 (비트코인, 이더리움, 코스피, 나스닥, 금)
- 환율 기준 외화투자 ROI 계산
- 리스크 대비 수익률 (샤프 지수 기반 효율 등급)
- **인사이트**: 포트폴리오 다각화에 도움, 과거 회상적이지만 미래 투자 결정에는 제한적
- **참고**: 구현 난이도 매우 높음 (4~6주 소요, 다수 API 연동 필요)

### 2.4 출력 및 시각화

#### 2.4.1 텍스트 출력 원칙

- 전문 용어 최소화
- 구체적인 숫자와 함께 의미를 연결하는 문장 사용
- 예시:
  - ❌ "CAGR이 5%입니다."
  - ✅ "지금처럼 투자하면 10년 후 자산이 약 1.6배가 됩니다. 이는 매년 5%씩 늘어나는 속도예요."

#### 2.4.2 시각화 요구사항

- **필수 그래프**:
  - 연도별 자산 변화 추이 (라인 차트)
  - 소득 vs 지출 비교 (바 차트)
  - 시나리오별 비교 (다중 라인 차트)
- **선택 그래프**:
  - 자산 구성 비율 (파이 차트)
  - 위험도 지표 (게이지 차트)
  - 목표 달성 진행도 (프로그레스 바)

#### 2.4.3 결과 다운로드

- **Streamlit**:
  - PDF 리포트 생성 및 다운로드
  - CSV 데이터 다운로드
  - 이미지(그래프) 다운로드
- **Notion**:
  - 데이터베이스에 자동 저장
  - 리포트 페이지 자동 생성

## 3. 비기능 요구사항

### 3.1 성능

- 입력 데이터 처리 시간: 3초 이내
- API 호출 실패 시 graceful degradation (기본값 사용)
- 그래프 렌더링: 5초 이내

### 3.2 사용성

- 초기 설정 가이드 제공
- 입력 필드에 도움말/예시 표시
- 에러 메시지: 사용자 친화적 문구 사용
- 반응형 디자인 (모바일 최소 지원)

### 3.3 데이터 관리

- **Streamlit (로컬 실행)**:
  - 세션 기반 데이터 저장 (브라우저 종료 시 삭제)
  - 결과 다운로드 기능으로 데이터 보존
- **Streamlit (Cloud 배포)**:
  - 세션 기반 데이터 저장 (브라우저 종료 시 삭제)
  - 결과 다운로드 기능 (JSON/CSV/PDF)
  - 브라우저 로컬 스토리지 활용 가능 (선택)
- **Notion**:
  - 데이터베이스 영구 저장
  - 사용자별 개별 데이터베이스
  - 시계열 데이터 추적 가능

### 3.4 보안 및 개인정보

- 개인 금융 정보 로컬 처리 (서버 전송 최소화)
- API 키는 환경 변수로 관리
- 개인정보 수집 최소화 (필수 입력만)

### 3.5 호환성

- **Streamlit**: Python 3.8 이상
- **Notion**: Notion API v1 이상
- 브라우저: Chrome, Edge, Safari 최신 2개 버전

## 4. 기술 스택

### 4.1 Streamlit 버전

- **프레임워크**: Streamlit
- **주요 라이브러리**:
  - pandas: 데이터 처리
  - plotly 또는 matplotlib: 시각화
  - requests: API 호출
  - python-dotenv: 환경 변수 관리
- **배포 플랫폼**:
  - Phase 1: Streamlit Cloud (추천) 또는 Termux + ngrok
  - Phase 2: Streamlit Cloud (프로덕션)

### 4.2 Notion 버전

- **Notion API**: 공식 Notion API (v1)
- **템플릿 구조**: 데이터베이스 + 페이지 템플릿
- **자동화**: Python 스크립트로 데이터 동기화 (선택)
- **배포**: 템플릿 공유 링크 또는 워크스페이스 복사

### 4.3 데이터 소스

- 한국은행 경제통계시스템 (ECOS) API
- CoinGecko API (암호화폐)
- 공개 금융 데이터 API

## 5. 개발 단계

### 5.1 Phase 1: MVP (프로토타입)

- **기간**: 추정 4~6주
- **기능**: 우선순위 1, 2번 카테고리
- **플랫폼**: 접근 가능한 배포 방식 (아래 옵션 참조)
- **목표**: 핵심 기능 검증 및 사용자 피드백 수집, 결과물 공유 가능

#### 5.1.1 Phase 1 배포 옵션

**요구사항**: Phase 1 결과물을 로컬이 아닌 곳에서도 접근 가능해야 함

**옵션 A: Streamlit Cloud 배포 (추천)**

- **방법**: GitHub 연동 후 Streamlit Cloud에 무료 배포
- **장점**:
  - 무료 (공개 저장소 기준)
  - URL로 바로 접근 가능
  - 자동 업데이트 (GitHub 푸시 시)
  - 가장 간단하고 빠른 배포
- **단점**:
  - 공개 저장소 필요 (프라이빗은 유료)
  - API 키 노출 주의 필요
- **구현 난이도**: 매우 쉬움
- **접근성**: 최고 (어디서나 URL로 접근)

**옵션 B: Termux + ngrok (모바일/로컬 서버)**

- **방법**: Android Termux에서 Streamlit 실행 후 ngrok으로 터널링
- **장점**:
  - 모바일에서도 실행 가능
  - 로컬 데이터 보안 (서버 없음)
  - 무료
- **단점**:
  - ngrok 무료 버전은 세션당 2시간 제한
  - 매번 URL 변경됨
  - Termux 설정 필요
- **구현 난이도**: 중간
- **접근성**: 중간 (ngrok URL 공유 필요, 제한적)

**옵션 C: Notion 템플릿 중심**

- **방법**: Notion 데이터베이스 + 수식 기반 계산
- **장점**:
  - 데이터 영구 저장
  - 배포 불필요
  - 접근성 좋음 (Notion 앱/웹)
- **단점**:
  - 복잡한 계산 제약 (Notion 수식 한계)
  - API 자동 연동 불가
  - 시각화 제한적
- **구현 난이도**: 쉬움 (하지만 기능 제한)
- **접근성**: 좋음 (Notion 접근 가능한 곳 어디서나)

**옵션 D: 하이브리드 (Streamlit Cloud + Notion)**

- **방법**: Streamlit Cloud로 계산/시각화, Notion으로 데이터 저장
- **장점**:
  - 각 플랫폼의 장점 활용
  - 접근성 + 데이터 영구 저장
- **단점**:
  - 두 플랫폼 관리 필요
  - Notion API 연동 필요 (선택적)
- **구현 난이도**: 중간
- **접근성**: 최고

#### 5.1.2 Phase 1 배포 전략 결정

**추천: 옵션 A (Streamlit Cloud)**

이유:

1. 가장 빠르고 간단한 배포
2. URL 공유로 즉시 접근 가능
3. 결과물 공유 및 피드백 수집 용이
4. Phase 2로 자연스럽게 확장 가능

**대안: 옵션 D (하이브리드)**

데이터 영구 저장이 Phase 1에서도 중요하다면:

- Streamlit Cloud: 계산 및 시각화
- Notion 템플릿: 데이터 입력 및 결과 저장
- 사용자 가이드: Streamlit에서 계산 → Notion에 결과 복사

### 5.2 Phase 2: 확장

- **기능**: 우선순위 3~10번 카테고리 점진적 추가
- **플랫폼**: Streamlit Cloud 웹 배포 + Notion 템플릿 배포
- **목표**: 완전한 기능 제공 및 대중 배포
- **추가 기능**:
  - Notion API 자동 연동 (선택)
  - 사용자 인증 (선택)
  - 데이터 백업 기능

## 부록: 구현 난이도 평가 기준

### 평가 기준

- **쉬움**: 입력 데이터 기반 단순 계산, 외부 API 불필요, 예상 개발 시간 1주 이내
- **중간**: 외부 API 필요하지만 공개 API 사용 가능, 계산 복잡도 중간, 예상 개발 시간 2~3주
- **어려움**: 복잡한 알고리즘, 다수 API 연동, 머신러닝/통계 모델 필요, 예상 개발 시간 4주 이상

### 평가 요소

각 카테고리는 다음 요소를 종합하여 평가됨:

1. **데이터 의존성**: 외부 API 필요 여부, 데이터 수집 난이도
2. **계산 복잡도**: 수학적 모델의 복잡성, 알고리즘 난이도
3. **기술적 도전**: 새로운 기술 스택 필요 여부, 통합 난이도
4. **개발 시간**: 예상 개발 및 테스트 기간
5. **유지보수성**: 지속적인 데이터 업데이트 필요성
6. **인사이트 제공 능력**: 사용자에게 실질적이고 실행 가능한 인사이트를 제공할 수 있는가?

#### 인사이트 제공 능력 평가 기준

- **높음**: 계산 결과가 즉시 실행 가능한 행동으로 이어질 수 있음 (예: "지출을 10% 줄이면 5년 후 XXX만원 더 모을 수 있습니다")
- **중간**: 결과 해석이 필요하지만 유용한 정보 제공 (예: "현재 자산의 실질가치는 XXX만원입니다")
- **낮음**: 단순 계산 결과만 제공, 추가 해석 필요 (예: "CAGR은 5%입니다")

**난이도**: **중간**

**구현 복잡도 요소**:

- 과거 데이터 조회 및 저장 (캐싱 필요)
- 실질가치 계산 공식 구현

**기술적 도전 과제**:

- API 인증 및 호출 제한 처리

**의존성**: 한국은행 API 또는 통계청 공개 데이터

**인사이트 제공 능력**: **중간**

- 실질가치 계산 결과는 유용하지만 즉시 실행 가능한 행동으로 이어지기 어려움
- "인플레이션만 반영했다면 vs 실제 성장률" 비교는 투자 의사결정에 도움
- 미래 구매력 예측은 장기 계획 수립에 유용하지만 단기 행동 변화로는 이어지기 어려움

**예상 개발 시간**: 1주

**기술적 도전 과제**:

- 시나리오별 비교 UI/UX 설계

**난이도**: **어려움**

**필요 기술/데이터**:

- 실시간 주가 API (코스피, 나스닥)
- 암호화폐 가격 API (CoinGecko, Binance 등)
- 과거 가격 데이터 (최소 5년)
- 환율 API
- 금 가격 API (선택)

**구현 복잡도 요소**:

- 다수 API 연동 및 통합

- 다양한 자산군 비교 알고리즘
- 샤프 지수 등 고급 금융 지표 계산
- API 호출 제한 및 에러 처리

**예상 개발 시간**: 4~6주

**기술적 도전 과제**:

- API 호출 제한 관리 (Rate Limiting)
- 과거 데이터의 정확도 및 보정
- 실시간 데이터 vs 과거 데이터 일관성
- 샤프 지수 계산을 위한 변동성 데이터

**의존성**:

- 주가 API (야후 파이낸스, Alpha Vantage 등)
- 암호화폐 API (CoinGecko, CoinMarketCap)
- 환율 API
- 금 가격 API

**인사이트 제공 능력**: **중간**

- "5년 전 비트코인을 샀다면 지금 XXX만원" → 과거 회상적이지만 미래 투자 결정에는 제한적
- 다양한 자산군 비교 → 포트폴리오 다각화에 도움
- 샤프 지수 기반 효율 등급 → 투자 전략 개선에 활용 가능하나 전문 지식 필요

#### 4. 부동산 기반 분석

**난이도**: **중간**

**필요 기술/데이터**:

**구현 복잡도 요소**:

- 대출 상환 계산 로직
- Break-even point 계산
- 지역별 상승률 적용

**예상 개발 시간**: 2~3주

**기술적 도전 과제**:

- 부동산 데이터 공개 API 부족 (수동 입력 의존 가능)

- 대출 상환 시뮬레이션 복잡도

**의존성**:

**인사이트 제공 능력**: **높음**
